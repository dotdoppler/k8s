# Kubernetes 系统学习路线图 (Java 后端开发者专属)

作为一名熟悉 Java 和 Docker 的后端开发者，我们学习 K8s 的目的是**为了更好地设计和部署微服务架构**。这份路线图将结合你现有的知识体系（Spring Boot, Docker），带你从零实操并深入理解 Kubernetes。

## 阶段一：从 Docker 到 Kubernetes 的思维转换（基础概念）
**目标**：理解为什么需要 K8s，以及 K8s 是如何对容器进行编排的。

1. **架构与组件**：
   - K8s 集群架构：Control Plane (API Server, ETCD, Scheduler, Controller Manager) 与 Node (Kubelet, Kube-proxy, Container Runtime)。
   - 对比 Docker Swarm，理解 K8s 的声明式 API 理念。
2. **核心资源抽象**：
   - `Pod`：K8s 的最小调度单元（为什么不用容器而是用 Pod？）。
   - `Deployment` & `ReplicaSet`：无状态应用的部署与扩缩容。
   - `Service`：应用在集群内部的服务发现与负载均衡。

## 阶段二：实操落地与本地集群搭建（动手实战）
**目标**：搭建本地实验环境，并让第一个 Java 应用跑在 K8s 上。

1. **环境搭建**：
   - 使用 `Minikube` 或 `Kind` (Kubernetes in Docker) 在 Mac 本地搭建单节点测试集群。
   - 熟悉 `kubectl` 命令行工具的日常使用。
2. **Java 应用容器化复习与进阶**：
   - 编写适用于 K8s 的 Spring Boot `Dockerfile`。
   - JVM 容器感知（Java 11/17/21 中的容器资源限制问题）。
3. **初次部署**：
   - 编写第一份 YAML 文件配置，部署 Spring Boot 应用。

## 阶段三：配置管理与网络路由（开发进阶）
**目标**：解决应用的外部访问和环境配置脱敏问题。

1. **应用配置外部化**：
   - `ConfigMap`：管理 application.yml 等非敏感配置。
   - `Secret`：管理数据库密码、证书等敏感信息。
2. **服务暴露与网关**：
   - `Service` 详解：ClusterIP, NodePort, LoadBalancer。
   - `Ingress`：7层 HTTP/HTTPS 路由规则（K8s 中的 "Nginx 网关"）。
3. **数据持久化 (Storage)**：
   - 为什么 Pod 销毁了数据会丢失？
   - `Volume`, `PersistentVolume (PV)`, `PersistentVolumeClaim (PVC)` 实践机制。

## 阶段四：高级应用编排与架构解耦（专家之路）
**目标**：处理复杂的有状态中间件应用，了解集群高可用调度。

1. **有状态应用编排**：
   - `StatefulSet`：如何在上 K8s 部署 MySQL、Redis Cluster 或 Kafka。
   - `Headless Service` 与稳态网络标识。
2. **节点守护与批处理**：
   - `DaemonSet`：日志收集 (Fluentd/Filebeat) 和监控组件。
   - `Job` & `CronJob`：定时任务处理。
3. **Java 微服务云原生改造指南**：
   - **健康检查**：`Liveness` 和 `Readiness` 探针结合 Spring Boot Actuator 的最佳实践。
   - **平滑重启**：优雅停机 (Graceful Shutdown) 与 K8s `PreStop` Hook。
   - **资源分配**：`Resource Requests & Limits` 对 JVM 堆内存（Xms/Xmx）的影响。

## 阶段五：生态扩展与发布系统（架构视角）
**目标**：提升交付效率，向架构师层面迈进。

1. **包管理工具 Helm**：
   - 像使用 Maven 一样管理 K8s 应用包。
   - 构建自己的 Chart 模板。
2. **集群调度与容灾**：
   - 污点 (`Taint`) 与容忍度 (`Toleration`)。
   - 节点亲和性 (`NodeAffinity`) 与 Pod 反亲和性。
3. **初识 Operator 控制器模式 (扩展 K8s 边界)**

## 项目目录结构规划
我们可以在当前 `workspace/k8s` 项目下采用如下目录结构来组织笔记和实操代码：

```text
k8s/
├── docs/                 # 放置学习笔记 (Markdown)
│   ├── 00-路线图.md       # 本文件
│   ├── 01-基础概念.md
│   ├── 02-实战部署.md
│   └── ...
├── apps/                 # 放置实操用的 Java 微服务源码
│   └── demo-service/     # Spring Boot Demo
├── manifests/            # 放置所有 K8s YAML 配置清单
│   ├── basic/            # 基础 Pod/Deployment 配置
│   ├── config/           # ConfigMap/Secret 配置
│   └── advanced/         # StatefulSet 等复杂配置
└── scripts/              # 存放本地集群搭建、清理等 Shell 脚本
    └── setup-cluster.sh
```
